# fridge.ai — Project Manifest

> 가정 맞춤형 냉장고 재료 폐기 최소화 엔진
> *"버리는 재료 없이, 오늘 뭐 먹지?를 해결한다."*

- **상태:** 아이디어 / 기획 단계 (MVP 프로토타입 진행 중)
- **플랫폼:** 웹 앱 (모바일 우선)
- **작성일:** 2026-06-14
- **갱신일:** 2026-06-15
- **문서 버전:** 0.3 (Team-Review Updated)

---

## 1. 비전 (Vision)

가정에서 발생하는 식재료 낭비를 구조적으로 줄인다. 사용자는 보유 재료를 손쉽게 등록하고,
시스템은 유통기한을 실시간으로 추적하며, 마감 임박 재료를 우선 소진할 수 있는 최적의 요리를
큐레이션한다. 매달 절약된 식비와 줄어든 탄소 발자국을 가시화함으로써
**음식물 쓰레기 감소 → 식비 절감 → 환경 기여**를 동시에 달성한다.

## 2. 문제 정의 (Problem)

- 2~3인 이상 가구는 식재료를 대량·묶음 구매하지만, 구성원 간 냉장고 현황이 공유되지 않는다.
- 냉장고 속 재료를 '망각'하여 멀쩡한 식재료를 통째로 폐기하는 일이 반복된다.
- "오늘 뭐 먹지"를 결정할 때 보유 재료 및 유통기한을 고려하지 못한다.
- 기존 레시피 앱은 콘텐츠 양은 많지만 유통기한 관리·폐기 분석 기능이 전무하다.
- 기존 메모·재고 앱은 기능이 단순해 요리 추천 연계가 불가능하다.
- 입력이 번거로워 지속 사용률이 낮다.

> **MVP 범위 단서:** 구성원 간 공유 미흡이 핵심 문제이나, MVP는 **단일 사용자**로 핵심 루프(등록→D-day→소진 추천)를 먼저 검증하고, **가구 공유는 확장 단계(Phase 3) 목표**로 둔다. (§10, F9)

## 3. 핵심 가치 제안 (Value Proposition)

1. **간편 등록** — 자주 사는 재료 버튼 + 카테고리 탭 + 재료 마스터 기반 검색·자동완성으로 입력 마찰 최소화. (재료는 마스터 목록에서 선택)
2. **D-day 자동 관리** — 유통기한 기반 D-day를 시스템이 실시간으로 추적·알림. 상태 구간: **임박(D-2 이하) · 주의(D-3~D-5) · 신선(D-6 이상)**.
3. **소진 우선 큐레이션** — 마감 임박 재료를 활용하는 요리를 요리/메뉴 DB에서 매칭해 YouTube·블로그 링크로 추천.
4. **환경·절약 리포트** — 월간 폐기 절감량을 탄소 환산값과 절약 식비로 가시화.

## 4. 경쟁 포지셔닝 (Competitive Positioning)

| 구분 | 기존 레시피 앱 | 기존 메모 앱 | **fridge.ai** |
|------|-------------|------------|--------------|
| 레시피 | 풍부 | 없음 | YouTube·블로그 링크 |
| 유통기한 관리 | 없음 | 단순 기록 | D-day 추적 + 알림 |
| 폐기 분석 | 없음 | 없음 | 월간 절감·환경 리포트 |
| 추천 기준 | 인기·취향 | 없음 | **임박도 우선** |

### 비고객 3계층 공략 전략

- **1계층 (입력 이탈 위험군):** 자주 사는 재료 버튼으로 등록 마찰 제거
- **2계층 (앱 관리 거부층):** "오늘 안 먹으면 버리는 요리" 직관적 제시
- **3계층 (환경 무관심 미개척층):** 식비 절약·환경 기여 리포트로 신규 유입

## 6. 추천 엔진 개념 (Curation Engine)

### 레시피 소싱 전략 (요리/메뉴 DB 기반)

저작권 문제를 피하기 위해 레시피 본문(조리법 전문)을 직접 소유하지 않는다.
대신 **오픈소스 데이터에서 수집한 "요리/메뉴 DB"**를 구축한다. 이 DB는 조리법 전문이 아니라
**요리명 ↔ 사용 재료 매핑(메타데이터)**만 저장한다. 사용자의 보유·임박 재료를 이 DB와 매칭해
적합한 요리를 선정하고, 각 요리에 대해 YouTube·블로그 링크를 추천한다.

```
보유 재고: [감자 D-2, 양파 D-5, 달걀 D-10]
→ 요리/메뉴 DB 매칭: 감자전(감자,양파) · 감자조림(감자,양파,달걀) …
→ 스코어링으로 정렬 (임박 재료 소진 우선)
→ 요리별 YouTube·블로그 링크 카드 표시
```

### 스코어링 (요리 추천 우선순위)

요리/메뉴 DB가 각 요리의 재료 구성을 보유하므로 아래 공식이 **실제로 계산 가능**해진다.

```
score(recipe) =
      w1 * (사용 재료 중 임박 재료 비율)
    + w2 * (보유 재료 매칭률)
    + w3 * (임박도 가중합, D-day 작을수록 ↑)
    - w4 * (부족 재료 수)
```

- **MVP 전략:** 규칙 기반 매칭·스코어링 → 데이터 축적 후 개인화·ML 고도화
- **장점:** 조리법 전문 미소유로 저작권 리스크 최소화, 스코어링 정밀 추천 가능, 외부 링크로 최신 콘텐츠 연결
- **선결 과제:** 요리/메뉴 DB의 **오픈소스 데이터 출처·라이선스 확보** (→ §14)
- **참고:** 외부 링크는 MVP에서 YouTube·블로그 검색 URL로 시작하고, 정밀 카드(썸네일·제목)는 YouTube Data API를 Phase 2에서 선택 도입

## 7. 환경·절약 리포트 (Report)

감성적 톤: **환경 기여 중심** ("이번 달 탄소 1.2kg를 줄였어요")

| 항목 | 내용 |
|------|------|
| 조리 완료 기록 | 추천 요리 → 실제 조리 완료 이력 |
| 폐기 기록 | 유통기한 초과 폐기 이력 및 금액 환산 |
| 월간 절감량 | 폐기 감소 재료의 탄소 발자국 환산값 |
| 식비 절약 | 소진 재료 금액 기준 절약액 추정 |
| 환경 기여 분석 | 누적 탄소 절감 트렌드 시각화 |

**데이터 출처 현황**
- ✅ **탄소 환산:** 재료 마스터에 `carbon_per_100g` 확보 (출처: 농촌진흥청 국가표준식품성분표 + 탄소 발자국 문헌). §14-Q2 해소.
- ⚠️ **식비 절약(단가):** 재료별 단가 데이터 **미확보** — 마스터에 가격 필드 없음. 절약액 추정의 선결 과제 (→ §14).

## 8. 사용자 (Users)

- **1차 타깃:** 2~3인 이상 집밥형 가구 — 식재료를 대량·묶음 구매하나 구성원 간 냉장고 정보가 공유되지 않아 망각·폐기가 잦은 그룹
- **2차 타깃:** 식비·환경에 관심 있는 가정 — 절약 리포트로 유입되는 신규 사용자
- **핵심 페르소나:** "장은 봤는데 냉장고에 뭐가 있는지 가족끼리 아무도 모르고, 결국 멀쩡한 걸 버리는 사람"

## 9. 기술 방향 (Tech Direction — 잠정)

> 기획 단계이므로 확정이 아니며, 설계 단계에서 재검토한다.

- **프론트엔드:** **MVP는 정적 HTML/CSS/바닐라 JS 프로토타입**(모바일 우선, localStorage). 백엔드·계정·DB 연동 시점에 **Next.js(React)로 전환**.
- **백엔드/데이터:** API 서버 + 관계형 DB (재료 마스터·재고·폐기 이력·요리/메뉴 DB)
- **인증:** 이메일/소셜 로그인 (백엔드 도입 단계)
- **재료 마스터:** 카테고리 10종 · 약 131종 재료 (단위·`weight_per_unit_g`·`carbon_per_100g` 포함). 등록은 마스터에서만 선택, 미등록은 신규 요청 큐(F10).
- **레시피 소싱:** 오픈소스 기반 **요리/메뉴 DB**(요리↔재료 매핑) + YouTube·블로그 링크. YouTube Data API는 Phase 2 선택 도입.
- **추천 엔진:** 규칙 기반 매칭·스코어링 모듈 (요리/메뉴 DB 활용)
- **유통기한 데이터:** 사용자 직접 입력 (MVP) → 카테고리 프리셋 추후 추가

## 10. 범위 (Scope)

### MVP에 포함 (In Scope)
- 재료 등록 — 재료 마스터(약 131종) 기반 선택 입력, 카테고리 자동 지정 (F1)
- 유통기한 D-day 추적 및 재고 대시보드(상태 필터 포함) (F3–F4)
- 요리/메뉴 DB 구축 + 매칭·스코어링 기반 소진 우선 레시피 추천(YouTube·블로그 링크) (F5)
- **단일 사용자** 기준 (가구 공유는 Phase 3)

### 2차 출시 (Phase 2)
- 자주 사는 재료 버튼 (F2)
- 조리 완료·폐기 기록 (F6)
- 만료 알림 (F7)
- 식비 절약·환경 리포트 (F8)

### 초기 범위 제외 (Out of Scope — 추후)

## 12. 성공 지표 (Success Metrics)

- **북극성 지표:** 사용자당 월간 폐기 재료 수 감소율 — *측정:* (전월 폐기 건수 − 당월 폐기 건수) / 전월 폐기 건수

- 추천 레시피 클릭/조리 전환율 (추천 노출 → 링크 클릭 → 조리 완료)
- 월간 환경 리포트 확인율 (3계층 유입 지표)
