# fridge.ai — Project Manifest

> 가정 맞춤형 냉장고 재료 폐기 최소화 엔진
> *"버리는 재료 없이, 오늘 뭐 먹지?를 해결한다."*

- **상태:** MVP 프로토타입 완성 (Demo Day 2026-06-19)
- **플랫폼:** 웹 앱 (모바일 우선)
- **작성일:** 2026-06-14
- **갱신일:** 2026-06-19
- **문서 버전:** 0.4 (Demo Day 반영)

---

## 1. 비전 (Vision)

가정에서 발생하는 식재료 낭비를 구조적으로 줄인다. 사용자는 보유 재료를 손쉽게 등록하고,
시스템은 유통기한을 실시간으로 추적하며, 마감 임박 재료를 우선 소진할 수 있는 최적의 요리를
큐레이션한다. 매달 절약된 식비와 줄어든 탄소 발자국을 가시화함으로써
**음식물 쓰레기 감소 → 식비 절감 → 환경 기여**를 동시에 달성한다.

## 2. 문제 정의 (Problem)

- 2~3인 이상 가구는 식재료를 대량·묶음 구매하지만, 구성원 간 냉장고 현황이 공유되지 않는다.
- 냉장고 속 재료를 '망각'하여 멀쩡한 식재료를 통째로 폐기하는 일이 반복된다.
- "오늘 뭐 먹지"를 결정할 때 보유 재료 및 유통기한을 고려하지 못한다.
- 기존 레시피 앱은 콘텐츠 양은 많지만 유통기한 관리·폐기 분석 기능이 전무하다.
- 기존 메모·재고 앱은 기능이 단순해 요리 추천 연계가 불가능하다.
- 입력이 번거로워 지속 사용률이 낮다.

> **MVP 범위 단서:** 구성원 간 공유 미흡이 핵심 문제이나, MVP는 **단일 사용자**로 핵심 루프(등록→D-day→소진 추천)를 먼저 검증하고, **가구 공유는 확장 단계(Phase 3) 목표**로 둔다. (§10, F9)

## 3. 핵심 가치 제안 (Value Proposition)

1. **간편 등록** — 자주 사는 재료 버튼 + 카테고리 탭 + 재료 마스터 기반 검색·자동완성으로 입력 마찰 최소화. (재료는 마스터 목록에서 선택)
2. **D-day 자동 관리** — 유통기한 기반 D-day를 시스템이 실시간으로 추적·알림. 상태 구간: **임박(D-2 이하) · 주의(D-3~D-5) · 신선(D-6 이상)**.
3. **소진 우선 큐레이션** — 마감 임박 재료를 활용하는 요리를 요리/메뉴 DB에서 매칭해 YouTube·블로그 링크로 추천.
4. **환경·절약 리포트** — 월간 폐기 절감량을 탄소 환산값과 절약 식비로 가시화.
5. **Proactive AI Guardian** — 임박 재료 조합을 AI가 설명하는 맞춤 배너로 능동적 냉장고 관리 유도.

## 4. 경쟁 포지셔닝 (Competitive Positioning)

|------|-------------|------------|--------------|
| 레시피 | 풍부 | 없음 | YouTube·블로그 링크 |
| 유통기한 관리 | 없음 | 단순 기록 | D-day 추적 + 알림 |

| AI 능동 안내 | 없음 | 없음 | **Proactive Guardian 배너** |

### 비고객 3계층 공략 전략

- **1계층 (입력 이탈 위험군):** 자주 사는 재료 버튼으로 등록 마찰 제거
- **2계층 (앱 관리 거부층):** "오늘 안 먹으면 버리는 요리" 직관적 제시
- **3계층 (환경 무관심 미개척층):** 식비 절약·환경 기여 리포트로 신규 유입

## 6. 추천 엔진 개념 (Curation Engine)

### 레시피 소싱 전략 (요리/메뉴 DB 기반)

저작권 문제를 피하기 위해 레시피 본문(조리법 전문)을 직접 소유하지 않는다.
대신 **오픈소스 데이터에서 수집한 "요리/메뉴 DB"**를 구축한다. 이 DB는 조리법 전문이 아니라
**요리명 ↔ 사용 재료 매핑(메타데이터)**만 저장한다. 사용자의 보유·임박 재료를 이 DB와 매칭해
적합한 요리를 선정하고, 각 요리에 대해 YouTube·블로그 링크를 추천한다.

```
보유 재고: [감자 D-2, 양파 D-5, 달걀 D-10]
→ 요리/메뉴 DB 매칭: 감자전(감자,양파) · 감자조림(감자,양파,달걀) …
→ 스코어링으로 정렬 (임박 재료 소진 우선)
→ 요리별 YouTube·블로그 링크 카드 표시
```

### 스코어링 (요리 추천 우선순위)

```
score(recipe) =
      35 × urgentRatio     (사용 재료 중 임박(D-2 이하) 재료 비율)
    + 15 × completeness    (보유 주재료 매칭률)
    + 40 × absBonus        (임박·주의 재료 절대 개수, 5개 상한 — 다재료 요리 우대)
    + 10 × seaRatio        (조미료·선택 재료 구비율)

필터 ①: (임박+주의) 주재료 ≥ 1개
필터 ②: 부족 주재료 ≤ 40%
```

- **MVP 전략:** 규칙 기반 매칭·스코어링 → 데이터 축적 후 개인화·ML 고도화
- **레시피 DB 소싱:** 식품안전나라 COOKRCP01 API(1,146건) + 내부 recipe_ingredients 매핑 병행
- **장점:** 조리법 전문 미소유로 저작권 리스크 최소화
- **참고:** 외부 링크는 MVP에서 YouTube·Naver 블로그 검색 URL로 시작, 정밀 카드는 Phase 2에서 YouTube Data API 도입

## 7. 환경·절약 리포트 (Report)

감성적 톤: **환경 기여 중심** ("이번 달 탄소 1.2kg를 줄였어요")

### 집계 공식 (MVP 구현)

```
wasteG     = (조리완료 건수 × 150g) + 부분사용 weight_g 합계
co2_kg     = wasteG × 0.00087
water_L    = wasteG × 0.2
envCost_원 = co2_kg × 10,000
```

| 항목 | 내용 |
|------|------|
| 구출한 재료 | 이번 달 조리완료 건수 (히어로 카드) |
| 음식물 쓰레기 절감량 | wasteG (g/kg 환산) + 쓰레기봉투 절약 수 |
| CO₂ 절감 | wasteG × 0.00087 kg |
| 물 절약 | wasteG × 0.2 L |
| 환경 비용 절감 | co2_kg × 10,000 원 |
| 조리 완료 | 조리완료 + 부분사용 합산 횟수 |
| 레벨 시스템 | 6단계 (냉장고 새내기 → 절약 새싹 → 절약 고수 → 냉장고 지킴이 → 냉장고 영웅 → 냉장고 마스터) |
| 주간 소진 추이 | 최근 7일 일별 스택 바 차트 (조리완료/부분사용 색상 구분, 오늘 하이라이트) |

**데이터 출처 현황**
- ✅ **탄소 환산:** 재료 마스터에 `carbon_per_100g` 확보 (출처: 농촌진흥청 국가표준식품성분표 + 탄소 발자국 문헌).
- ✅ **절감량:** 조리완료 건당 150g 추정 + 부분사용 직접 입력량 합산.
- ⚠️ **식비 절약(단가):** 재료별 단가 데이터 **미확보** — 절약 식비는 건당 2,500원 고정값으로 추정.

## 8. 사용자 (Users)

- **1차 타깃:** 2~3인 이상 집밥형 가구 — 식재료를 대량·묶음 구매하나 구성원 간 냉장고 정보가 공유되지 않아 망각·폐기가 잦은 그룹
- **2차 타깃:** 식비·환경에 관심 있는 가정 — 절약 리포트로 유입되는 신규 사용자
- **핵심 페르소나:** "장은 봤는데 냉장고에 뭐가 있는지 가족끼리 아무도 모르고, 결국 멀쩡한 걸 버리는 사람"

## 9. 기술 방향 (Tech Direction)

- **프론트엔드:** 정적 HTML/CSS/바닐라 JS 프로토타입 (모바일 우선). 백엔드·계정·DB 연동 시점에 **Next.js(React)로 전환**.
- **데이터 저장:** localStorage (STORE_KEY v5, LOG_KEY v2). 백엔드 도입 시 관계형 DB로 마이그레이션.
- **추천 엔진:** 규칙 기반 스코어링 모듈 (app.js 내). 레시피 DB: COOKRCP01 API(1,146건) + 내부 recipe_ingredients 매핑 병행.
- **소비기한:** 소비기한 직접 입력 + 유통기한→소비기한 자동 변환(expiry_ext_days 기준) 모드 전환 지원.
- **AI Guardian:** 임박 재료 조합별 사전 정의 설명문 + 동적 렌더링 (MVP). Phase 2에서 Claude/GPT API 연동.
- **배포:** GitHub Pages (main 브랜치 자동 배포).
- **인증:** 이메일/소셜 로그인 (백엔드 도입 단계).
- **재료 마스터:** 카테고리 10종 · 약 131종 재료 (단위·`weight_per_unit_g`·`carbon_per_100g`·`shelf_days`·`expiry_ext_days` 포함).
